请基于当前已经生成的“安全型团队提效 AI 工作台”原型，重新优化“工作台首页”这一个页面。

重要要求：
1. 这是对当前首页的重构优化，不是重做整套产品
2. 保留当前整体视觉风格、配色、导航结构和组件语言
3. 只优化“工作台首页”当前页面
4. 不要增加更多系统感，不要增加菜单，不要增加复杂筛选器
5. 目标是让首页从“轻量后台首页”进一步收口成“AI 行动工作台首页”

当前首页存在的问题：
- 还有一点传统看板首页感
- 信息块偏多，用户打开后先是在“看信息”，不是“马上开始处理事情”
- 输入区和快捷动作区有一定重复
- 高优先级事项、风险提醒、最近生成结果的层级还不够聚焦

请按以下方向重构首页：

一、首页目标重新定义
首页不是统计页，也不是传统仪表盘，而是“今天先做什么”的行动首页。
用户打开后，应该第一眼知道：
1. 我可以直接输入什么
2. 哪些事项需要我先确认
3. 当前最值得优先处理什么
4. 最近 AI 帮我产出了什么

二、页面结构重构要求
请将首页收敛为 4 个主区域，不要再做太多并列信息块：

1. 顶部主输入区（页面绝对核心）
要求：
- 成为首页最强视觉中心
- 一个大输入框，支持粘贴客户反馈、需求描述、会议内容或问题
- 占位文案自然，例如：
  “粘贴客户反馈、会议内容、需求描述或问题，我来帮你整理并推荐下一步”
- 输入区下方只保留 3 个核心动作按钮：
  - 开始处理
  - 发起影响分析
  - 生成会议纪要
- 不要再保留太多重复快捷按钮

2. 今日待确认（首页第二重点）
要求：
- 放在输入区下面，位置靠前
- 展示最需要人工确认的 3 到 5 条事项
- 每条信息只保留最关键字段：
  - 标题
  - 项目
  - 当前待确认内容
  - 推荐动作
- 强调“确认”动作，不要做成长列表
- 可以用卡片或简洁列表形式

3. 当前最值得关注（合并高优先级事项 + 风险提醒）
要求：
- 将原来“高优先级事项”和“风险提醒”合并成一个更聚焦的区域
- 页面里不要再分成两个同级大块
- 这里展示：
  - 高优先级事项
  - 关键风险提醒
- 每条卡片重点展示：
  - 标题
  - 优先级或风险等级
  - 当前状态
  - 推荐下一步
- 目标是帮助用户快速判断“先处理哪件事”

4. 最近 AI 生成结果（保留但降低权重）
要求：
- 展示最近生成的 PRD、测试点、会议纪要、上线说明
- 视觉权重低于“今日待确认”和“当前最值得关注”
- 更像轻量结果列表，不要做成大卡片墙
- 每条结果只展示：
  - 结果类型
  - 标题
  - 状态
  - 时间
  - 查看入口

三、必须删除或弱化的内容
1. 不要让首页出现过多重复快捷动作
2. 不要堆太多信息块
3. 不要使用复杂图表
4. 不要让首页更像项目管理看板
5. 不要增加新的复杂模块

四、交互要求
1. 用户进入首页后，主路径必须非常清楚：
   输入内容 → 待确认 → 当前重点事项 → 查看结果
2. 所有区块都要能继续进入下一步，但不要做过多跳转入口
3. 页面必须体现“行动优先”，不是“浏览优先”
4. 用户应明显感受到：
   - AI 在前面帮我接住事情
   - 我只要先确认、再推进

五、视觉要求
1. 保持当前企业级、克制、现代 SaaS 风格
2. 更轻、更聚焦、更少信息噪音
3. 顶部输入区必须成为唯一最强焦点
4. “今日待确认”要有明显的第二优先级
5. “最近 AI 生成结果”应明显更轻

六、示例数据要求
请继续使用真实中文示例数据，例如：
- 审批节点支持抄送
- 导出明细增加筛选条件
- 移动端登录超时优化
- 客户运维平台二期
- 流程审批重构

状态示例：
- 待确认
- 分析中
- 已生成
- 高优先级
- 中风险

最终目标：
请把当前首页优化成一个真正“轻、直接、先做事”的 AI 工作台首页，而不是传统后台首页。
